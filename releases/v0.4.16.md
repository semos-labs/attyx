# Attyx v0.4.16

## Sessions are on by default ([#293](https://github.com/semos-labs/attyx/pull/293))

New Attyx windows now start with daemon-backed sessions enabled. That means named sessions, persistence, session switching, and cross-session agent tooling work out of the box — no config edit required.

If you prefer the old direct-PTY mode, set:

```toml
[sessions]
enabled = false
```

Existing configs that already set `sessions.enabled = false` keep that behavior.

## Session switching is faster again ([#293](https://github.com/semos-labs/attyx/pull/293))

Switching between sessions no longer waits between grid-snapshot chunks. Large panes and busy TUIs should feel immediate again instead of pausing for a few hundred milliseconds.

## Better agent dashboard layout ([#293](https://github.com/semos-labs/attyx/pull/293))

`attyx dashboard` now uses the available terminal width more intelligently, giving the tab column extra room instead of wasting space or truncating too aggressively.

The inline interaction panel also has more breathing room: it shows more of the selected agent's latest message, supports longer replies, and keeps the selected row visible while you move through larger agent lists.

## Agent effort is visible ([#293](https://github.com/semos-labs/attyx/pull/293))

Agent tables now show each agent's reasoning/model effort when available. The new **EFFORT** field appears in `attyx dashboard`, `attyx list agents`, and JSON agent output.

The dashboard also hides elapsed time for idle agents, so the timer only calls attention to agents that are actively working or waiting on you.
